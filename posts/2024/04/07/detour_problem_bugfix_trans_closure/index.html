Original "fix":

While typesetting a paper for my solution to the detour problem, I realized that the algorithm is still flawed after the previous bugfix.

Here is a test case that triggers the bug:

VDJ-122 returns true on edge 1 and false on edge 2; VDJ-221 returns true on edge 2 and false on edge 1. But no test is performed on the two edges combined because the backward "flight" has a "stopover"!

Fortunately a fix isn't too involved.

What we need is a preprocessing step like this:

Let $E' = \varnothing$
For $u,v \in \mathcal{B}_{s,t}$:
  If $dist(s,u) > dist(s,v)$ and
    $dist_{G-\mathcal{B}_{s,t}+u+v}(u,v) < \infty$:
    Add $(u,v)$ to $E'$
Let $E''$ be the transitive closure of $E'$
Replace $E$ with $E \cup E''$

After the preprocessing, whenever there is a backward hop, or a consecutive sequence of backward hops, there is a backward edge.

Nice!

Without the quick fix, we would need to worry about a problem like this:

Collision-free forward and backward paths on a layered DAG

Let $G = (V,E)$ be a DAG with the following properties:

There is a set $S \subset V$ of source nodes with no inbound edges.

There is a set $T \subset V$ of target nodes with no outbound edges.

There is $n \in \mathbb{N} - \{0\}$ such that all maximal paths in $G$ have length $n$. A path in $G$ is maximal if it cannot be further extended on either end.

For $u,v \in V$, let $dist(u,v)$ denote the edge count of a shortest path from $u$ to $v$ (or $\infty$ if no such path exists), and define $lvl(v) := min_{s \in S}dist(s,v)$ (therefore we have $0 \le lvl(v) \le n$).

Given $s, s' \in S$ and $t, t' \in T$ such that $dist(s, t) = dist(s', t') = n$, let $E' \subseteq \{(u,v) \in V \times V| lvl(u) > lvl(v) \}$ be a set of backward edges such that there is a path from $t'$ to $s'$ using $E'$ alone.

We'd like an efficient algorithm to decide if there is a path in $G$ from $s$ to $t$ and a path from $t'$ to $s'$ using only edges in $E'$ such that the two paths are vertex disjoint.

A naive solution would take roughly $\mathcal{O}(m^{2n})$, where $m$ denotes the maximum number of nodes in a level. A slightly better divide and conquer takes $\mathcal{O}(m^{2log(n)})$. There is no obvious further improvement in sight, yet it definitely doesn't look like we can get a reduction from 3SAT1 as in On the maximum disjoint paths problem on edge-colored graphs by Wu.

I think this might be an interesting problem in its own right so am leaving it here for posterity. "If you chop down a tree, plant a new one."
